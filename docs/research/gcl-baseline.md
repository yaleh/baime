## 关键发现

1. **E 是 GCL 的主导分量**（57.4%）：压缩 gate 负载的首要杠杆是减少 gate 判断所需读取的显性 artifact 内容（通过收窄判断范围或简化 DoD 结构）。

2. **C 是第二分量**（30.9%）：高耦合任务（多跨界引用）会显著提升 gate 负载。

3. **H 相对较小且稳定**（11.7%，标准差 0.71）：当前 BAIME 任务的隐性知识需求已被大部分外化。H 的均值为 1.70，意味着平均每次 gate 仍有约 1.7 个前提需要依靠判断者记忆或推断——这个量不随 artifact 增加而线性下降（见 gcl-intervention.md 的 H4 验证）。

4. **dod-eval 的低负载（GCL=5）验证了"收窄范围"的方向**：自动化 DoD 守卫通过把 gate 判断范围从"整个任务"收窄到"DoD 条目是否通过"，将 GCL 压缩到全集的 34%。

## 局限性

- **N=20**：样本量有限，分层分析（特别是 epic-evaluate: n=1, dod-eval: n=2）统计功效低。
- **H 估算偏差**：同一分析者同时估算 H 和构建分析框架，存在确认偏差风险。
- **任务历史范围**：语料仅覆盖 TASK-125、TASK-136–149，为 2026-06-21 至 2026-06-22 的两日窗口，可能不代表更长时间跨度的分布。
- **gate 类型混合**：不同 gate 类型的 GCL 结构差异大，混合均值的解释需谨慎。
